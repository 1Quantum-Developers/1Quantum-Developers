Reporting Guidelines

Thank you for taking the time to report an issue. These guidelines help us quickly understand, triage, and resolve reports — especially when evidence is provided clearly and concisely.

What to report

  • Security vulnerabilities or potential exploits
  • Abusive or illegal content or behavior
  • Policy violations (copyright, trademark, harassment)
  • Privacy or data-leak incidents
  • Other incidents or concerns affecting the project or users

What we need from you

When filing a report please include as many of the following as you can:

  • Category: (e.g. vulnerability, abuse, policy, privacy)
  • Summary: One-line description of the problem
  • Details: Full description of what you observed
  • Steps to reproduce: Exact steps, URLs, commands, and payloads
  • Evidence: Screenshots, logs, network captures, stack traces, timestamps
  • Affected targets: Accounts, repos, services, or endpoints involved
  • Severity estimate: Low / Medium / High / Critical
  • Contact info (optional): Email and name if you want follow-up

Example report template

Category: vulnerability
Summary: Unauthorized read access to private config on /api/config
Details: When sending GET /api/config with X-Experiment: 1 header the endpoint returns the private config for other users.
Steps to reproduce:
  1) curl -H "X-Experiment: 1" https://example.com/api/config?user=123
  2) Observe JSON with keys "secretKey"
Evidence: screenshot and curl output attached
Affected: /api/config endpoint, user IDs 120..125
Severity: high
Contact: hacker@example.com (optional)

How to submit

You can submit a report using the web form below. If you prefer email, see SECURITY.md for secure disclosure instructions for security vulnerabilities.

Submit a report

We do not accept large file attachments through this form. If you need to share large logs or captures, please contact us directly (see SECURITY.md) or provide a secure link.

Confidentiality and disclosure

We treat reports confidentially during triage. If your report is about a security vulnerability and you want a coordinated disclosure, please follow the instructions in SECURITY.md. We will not publish identifying information about reporters without consent.

What to expect

  • Acknowledgement: within 1–3 business days for most reports.
  • Triage and initial assessment: timeline depends on severity; urgent issues may be addressed immediately.
  • Follow-up: we may ask for additional evidence or permission to share more information.
